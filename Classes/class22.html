02 — Background

What is Supervised Fine-Tuning?

A base language model is essentially an autocomplete engine — given some text, it predicts the most likely next token. It has no concept of "following instructions." If you ask it "What is the capital of France?", it might respond with another question, or continue in essay form, or list more geography questions.

SFT teaches the model a new skill: given an instruction from a user, produce a helpful, relevant response. It does this by training on thousands of (instruction, response) pairs, where the model sees both the prompt and the ideal answer, and learns to predict the ideal answer given the prompt.

🧠

Knowledge → Behaviour

CPT fills the model's "brain" with domain knowledge. SFT teaches it to express that knowledge correctly in response to human prompts.

💬

Completion → Conversation

SFT adds conversational structure (system/user/assistant roles) that turns a raw text predictor into a chat participant.

🎯

Focused Learning

Unlike CPT where loss is computed over all tokens, SFT masks the prompt — the model only learns to predict the response tokens.

📊

Efficient Data Use

SFT needs far fewer examples than pre-training — typically 500–10,000 high-quality pairs are enough for meaningful behaviour change.

03 — Core Concepts

Key Concepts You Need to Know

📖 Key Terminology

Supervised Fine-Tuning (SFT)
Training a pre-trained model on labelled (prompt, response) pairs using the same next-token prediction objective, but with the loss masked to only the response tokens.
Instruction Dataset
A collection of (instruction, optional_input, response) or (conversation) examples used to teach a model to follow user directives.
Chat Template
A structured text format with special tokens that marks the start/end of messages and identifies who is speaking (system, user, assistant). Required for SFT and inference.
Loss Masking
Setting the loss to zero on prompt tokens so the model only learns to predict the response. Prevents the model from "memorising" the user's input rather than learning to answer.
Alpaca Format
A simple 3-field dataset format: instruction, input (optional context), output. Designed for single-turn instruction following. Named after Stanford's Alpaca project.
ShareGPT Format
A multi-turn conversation format with a "conversations" list of {from, value} pairs. Ideal for modelling back-and-forth dialogue.
ChatML
A structured format using special tokens <|im_start|> and <|im_end|> with explicit role labels. Native format for OpenAI-style chat models and widely adopted in the community.

Loss Masking — The Critical Difference

The most important technical distinction between CPT and SFT is where the loss is computed. Consider this training example:

Token-Level Loss Computation
CPT (all tokens): What is Paris ? Paris is the capital of France .
↑ Loss computed on ALL tokens (prompt + response)
SFT (masked): What is Paris ? Paris is the capital of France .
↑ Loss = 0 on prompt tokens (struck-through) · Loss computed ONLY on response tokens
Prompt token (user input) Response token (model output) — loss computed here Masked (loss = 0)
💡

Why mask the prompt? If you compute loss on both prompt and response, the model learns to "predict" the user's input — which is useless (we already know what the user said). By masking the prompt, you force the model to focus entirely on generating a good response given that input. This is what creates instruction-following behaviour.

Chat Templates

A chat template wraps each turn of a conversation in special tokens that the model uses to understand speaker identity and message boundaries. Without them, the model can't distinguish where the user's turn ends and its response begins.

Llama-3 template example 
<|begin_of_text|>
<|start_header_id|>system<|end_header_id|>

You are a helpful medical assistant.<|eot_id|>

<|start_header_id|>user<|end_header_id|>

What are the symptoms of appendicitis?<|eot_id|>

<|start_header_id|>assistant<|end_header_id|>

Appendicitis typically presents with...          ← model generates from here
<|eot_id|>
🟢

Consistency is critical: Whatever template you use during SFT training, you must use the exact same template at inference time. Mismatched templates cause the model to produce garbled or off-format outputs. Unsloth's get_chat_template() handles this automatically when you save and reload the tokenizer.

04 — Dataset Formats

Instruction Dataset Formats

Three formats dominate the SFT ecosystem. Choosing the right one depends on your task — single-turn Q&A vs multi-turn conversation.

Alpaca

Alpaca Format

Three fields: instruction, input (optional context), output. Simple, human-readable, single-turn only.

✓ Easy to create manually
✓ Huge community datasets
✓ Great for single-turn tasks
✗ No multi-turn support
✗ No system prompt field
ShareGPT

ShareGPT Format

A conversations list of {from, value} objects. Supports multi-turn, system prompts, and complex dialogue.

✓ Multi-turn dialogue
✓ Mirrors real ChatGPT interactions
✓ Widely available on HuggingFace Hub
✗ More complex to prepare
✗ Requires standardisation step
ChatML

ChatML Format

Uses <|im_start|> and <|im_end|> special tokens with explicit role labels. Native format for OpenAI-style models.

✓ Most explicit structure
✓ Industry standard for chat models
✓ Native Unsloth support
✗ Special tokens must be in tokenizer
✗ Less human-readable

Alpaca Format — In Detail

Raw JSON row
{
  "instruction": "Summarise the key symptoms of Type 2 Diabetes.",
  "input": "",
  "output": "Key symptoms include increased thirst, frequent urination, fatigue, blurred vision, and slow-healing wounds."
}
With context (input field)
{
  "instruction": "Classify the sentiment of this review.",
  "input": "The battery life is terrible and the screen cracked after one week.",
  "output": "Negative"
}

The Alpaca template used during training stitches these fields together:

Alpaca prompt template 
### Without input field:
Below is an instruction that describes a task.
Write a response that appropriately completes the request.

### Instruction:
{instruction}

### Response:
{output}<eos>

### With input field:
Below is an instruction that describes a task, paired with an input.
Write a response that appropriately completes the request.

### Instruction:
{instruction}

### Input:
{input}

### Response:
{output}<eos>

ShareGPT Format — In Detail

ShareGPT JSON row
{
  "conversations": [
    {
      "from": "system",
      "value": "You are a helpful coding assistant."
    },
    {
      "from": "human",
      "value": "Write a Python function to reverse a string."
    },
    {
      "from": "gpt",
      "value": "def reverse_string(s):\n    return s[::-1]"
    }
  ]
}
Multi-turn ShareGPT
{
  "conversations": [
    {"from": "human", "value": "What is LoRA?"},
    {"from": "gpt",   "value": "LoRA stands for Low-Rank Adaptation..."},
    {"from": "human", "value": "How does it save memory?"},
    {"from": "gpt",   "value": "Instead of updating all weights..."}
  ]
}

ChatML Format — In Detail

ChatML raw format 
<|im_start|>system
You are a helpful assistant.<|im_end|>
<|im_start|>user
Explain gradient descent in simple terms.<|im_end|>
<|im_start|>assistant
Gradient descent is like rolling a ball down a hill...
<|im_end|>
💬

Which format should you pick? For new single-turn datasets you build yourself, Alpaca is simplest. For conversational assistants or data sourced from ChatGPT exports, use ShareGPT. For maximum compatibility with community tooling and models, ChatML is the modern standard. Unsloth supports all three via get_chat_template().

05 — End-to-End Pipeline

SFT Training Pipeline — Step by Step

01

Curate & Clean Instruction Dataset

Collect or build (instruction, response) pairs. Aim for quality over quantity — 1,000 excellent examples beat 50,000 mediocre ones. Deduplicate. Filter pairs where the response is too short (<20 tokens) or too long (>2048 tokens). Remove personally identifiable information (PII).

02

Choose & Apply a Chat Template

Pick Alpaca, ShareGPT, or ChatML depending on your data structure. Use Unsloth's get_chat_template() to apply it consistently. The template adds special tokens and structures each sample into the system/user/assistant format.

03

Load Model with QLoRA

Same as CPT — use FastLanguageModel.from_pretrained() with 4-bit quantisation. For SFT, you can use either a base model (if you did CPT first) or an instruct model (for further specialisation). LoRA rank can be lower than CPT — 8 to 64 is typical.

04

Configure SFTTrainer with Loss Masking

TRL's SFTTrainer handles prompt masking automatically when you pass the formatted dataset. Key difference from CPT: do NOT use packing=True unless you have very short samples, as packing can mix documents and confuse the loss mask boundaries.

05

Train & Monitor Loss Curves

Watch both training loss and validation loss. Training loss should decrease; validation loss should also decrease and then plateau. If validation loss rises while training loss falls, you're overfitting — stop early and reduce epochs.

06

Chat-Style Inference

After training, use the model in chat mode — always apply the tokenizer's apply_chat_template() to format your prompt before passing it to the model. This is what distinguishes SFT inference from CPT inference (which was plain text completion).

07

Save, Merge, and Push to Hub

Save the LoRA adapter. Optionally merge it into the base model for deployment. Push to HuggingFace Hub. Consider exporting to GGUF format for local inference with llama.cpp or Ollama.

Step 4–5 in Detail — SFT Training Configuration

python · SFT training 
from trl import SFTTrainer
from transformers import TrainingArguments
from unsloth.chat_templates import train_on_responses_only

# ── Format dataset: apply template to each row ──────────────────
def formatting_func(examples):
    # For Alpaca format
    texts = []
    alpaca_prompt = """Below is an instruction. Write a response.

### Instruction:
{}

### Response:
{}"""
    for instruction, output in zip(examples["instruction"], examples["output"]):
        text = alpaca_prompt.format(instruction, output) + tokenizer.eos_token
        texts.append(text)
    return {"text": texts}

formatted_dataset = dataset.map(formatting_func, batched=True)

# ── Training Arguments ───────────────────────────────────────────
training_args = TrainingArguments(
    output_dir = "./sft-output",
    num_train_epochs = 3,          # 2-5 epochs is typical for SFT
    per_device_train_batch_size = 2,
    gradient_accumulation_steps = 4,
    warmup_ratio = 0.05,
    learning_rate = 2e-4,          # 1e-4 to 3e-4 for SFT
    fp16 = not torch.cuda.is_bf16_supported(),
    bf16 = torch.cuda.is_bf16_supported(),
    logging_steps = 25,
    save_steps = 200,
    evaluation_strategy = "steps",  # watch val loss!
    eval_steps = 100,
    load_best_model_at_end = True,
    optim = "adamw_8bit",
    lr_scheduler_type = "cosine",
    seed = 42,
    report_to = "none",
)

# ── Build trainer ────────────────────────────────────────────────
trainer = SFTTrainer(
    model = model,
    tokenizer = tokenizer,
    train_dataset = formatted_dataset,
    dataset_text_field = "text",
    max_seq_length = max_seq_length,
    dataset_num_proc = 2,
    packing = False,       # False for SFT to preserve prompt masks
    args = training_args,
)

# ── KEY: Mask the prompt tokens, train on responses only ─────────
trainer = train_on_responses_only(
    trainer,
    instruction_part = "### Instruction:\n",
    response_part    = "### Response:\n",
)

# ── Train! ───────────────────────────────────────────────────────
trainer.train()
⚠️

train_on_responses_only is the Unsloth helper that applies prompt masking. It scans each training sample for the instruction_part and response_part markers and sets loss to zero on all tokens before the response. Without this, you'd be doing CPT-style training, not SFT.

06 — Complete Example

Complete Minimal SFT Script

A full end-to-end notebook for Google Colab (T4 GPU) using the Alpaca-cleaned dataset.

python · complete SFT example 
## ═══════════════════════════════════════════════════
## CLASS 22 — Instruction Fine-Tuning (SFT)
## Supervised Fine-Tuning with Unsloth + QLoRA
## Platform: Google Colab (T4, 16GB VRAM)
## ═══════════════════════════════════════════════════

# CELL 1: Install
# !pip install unsloth datasets trl -q --break-system-packages

# CELL 2: Imports
from unsloth import FastLanguageModel
from unsloth.chat_templates import get_chat_template, train_on_responses_only
from datasets import load_dataset
from trl import SFTTrainer
from transformers import TrainingArguments, TextStreamer
import torch

# CELL 3: Config
MODEL_NAME    = "unsloth/llama-3-8b-bnb-4bit"
MAX_SEQ_LEN   = 2048
LORA_RANK     = 16
EPOCHS        = 3
LR            = 2e-4
BATCH_SIZE    = 2
GRAD_ACC      = 4
CHAT_TEMPLATE = "llama-3"

# CELL 4: Load model + LoRA
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name=MODEL_NAME, max_seq_length=MAX_SEQ_LEN, load_in_4bit=True)
model = FastLanguageModel.get_peft_model(
    model, r=LORA_RANK, lora_alpha=LORA_RANK,
    target_modules=["q_proj","k_proj","v_proj","o_proj",
                    "gate_proj","up_proj","down_proj"],
    lora_dropout=0, bias="none",
    use_gradient_checkpointing="unsloth", random_state=42)
tokenizer = get_chat_template(tokenizer, chat_template=CHAT_TEMPLATE)

# CELL 5: Load and format dataset
ALPACA_PROMPT = """Below is an instruction. Write a response.

### Instruction:
{}

### Response:
{}"""

dataset = load_dataset("yahma/alpaca-cleaned", split="train[:5000]")

def format_alpaca(batch):
    texts = []
    for instr, inp, out in zip(batch["instruction"],
                                 batch["input"], batch["output"]):
        if inp:
            text = ALPACA_PROMPT.format(instr + "\n\nContext:\n" + inp, out)
        else:
            text = ALPACA_PROMPT.format(instr, out)
        texts.append(text + tokenizer.eos_token)
    return {"text": texts}

dataset = dataset.map(format_alpaca, batched=True, remove_columns=dataset.column_names)

# CELL 6: Train-val split
split = dataset.train_test_split(test_size=0.05, seed=42)
train_ds, val_ds = split["train"], split["test"]

# CELL 7: Trainer
trainer = SFTTrainer(
    model=model, tokenizer=tokenizer,
    train_dataset=train_ds, eval_dataset=val_ds,
    dataset_text_field="text", max_seq_length=MAX_SEQ_LEN,
    packing=False, dataset_num_proc=2,
    args=TrainingArguments(
        output_dir="./sft_out", num_train_epochs=EPOCHS,
        per_device_train_batch_size=BATCH_SIZE,
        gradient_accumulation_steps=GRAD_ACC,
        learning_rate=LR, warmup_ratio=0.05,
        lr_scheduler_type="cosine", optim="adamw_8bit",
        bf16=torch.cuda.is_bf16_supported(),
        fp16=not torch.cuda.is_bf16_supported(),
        evaluation_strategy="steps", eval_steps=100,
        logging_steps=25, load_best_model_at_end=True,
        seed=42, report_to="none",
    ),
)
# Apply prompt masking!
trainer = train_on_responses_only(
    trainer,
    instruction_part="### Instruction:\n",
    response_part="### Response:\n",
)
trainer.train()

# CELL 8: Chat inference
FastLanguageModel.for_inference(model)
messages = [{"role":"user", "content":"Explain photosynthesis to a 10-year-old."}]
inputs = tokenizer.apply_chat_template(
    messages, tokenize=True, add_generation_prompt=True, return_tensors="pt").to("cuda")
TextStreamer(tokenizer, skip_prompt=True)
model.generate(input_ids=inputs, max_new_tokens=200,
               streamer=TextStreamer(tokenizer, skip_prompt=True))

# CELL 9: Save
model.save_pretrained("my-sft-model")
tokenizer.save_pretrained("my-sft-model")
07 — Practitioner Notes

Tips, Pitfalls & Best Practices

✅ Things that work well

  • ✓ Always call train_on_responses_only() — without prompt masking, you're doing CPT, not SFT.
  • ✓ Use train/validation split and monitor validation loss. Stop training when val loss plateaus, not at a fixed epoch count.
  • ✓ Quality > Quantity. 1,000 carefully written examples routinely outperform 100,000 scraped ones. Filter aggressively.
  • ✓ Save the tokenizer alongside your adapter — the chat template is stored in the tokenizer, not the model weights.
  • ✓ Use TextStreamer during inference to get streaming token output — useful for demos and checking quality mid-session.
  • ✓ If fine-tuning an instruction model (not base), use lower learning rates (1e-4 or below) to preserve existing instruction-following.

❌ Common mistakes

  • ✗ Forgetting to apply apply_chat_template() at inference time after using it during training — causes garbage output.
  • ✗ Mismatching the chat template between training and inference (e.g. training with Llama-3 template but inferring with ChatML).
  • ✗ Using packing=True without careful masking — document boundaries can bleed into each other and corrupt the loss mask.
  • ✗ Skipping a val split and running too many epochs — leads to overfitting where the model memorises training examples rather than generalising.
  • ✗ Including low-quality or inconsistent responses in your dataset — the model learns exactly what you show it, including the mistakes.
  • ✗ Not adding eos_token at the end of each sample — the model never learns when to stop generating and will hallucinate beyond the answer.

📐 Hyperparameter Reference for SFT

Hyperparameter Recommended Value Notes
LoRA rank (r) 8 – 64 Lower than CPT; 16 is a reliable default
Learning rate 1e-4 – 3e-4 Use 1e-4 when starting from an instruct model
Epochs 2 – 5 Stop when val loss plateaus (use early stopping)
Max seq length 2048 Match the longest response in your dataset + prompt
Packing False Keep False to preserve loss mask boundaries
Warmup ratio 0.03 – 0.1 5% of total steps is a safe default
Dataset size 1K – 50K samples Quality matters more than volume
08 — Self-Check

Quiz — Test Your Understanding

📝 Concept Check Questions

1. What is the defining technical difference between SFT and CPT in terms of how loss is computed?

▶ Show Answer
In CPT, loss is computed over all tokens (prompt and completion). In SFT, the prompt (instruction) tokens are masked — their loss is set to zero — and loss is computed only on the response tokens. This forces the model to learn how to generate a good response rather than memorising the input.

2. What is a chat template, and what happens if you use a different template at inference time than you used during training?

▶ Show Answer
A chat template is a structured format using special tokens (like <|im_start|>, <|eot_id|>) that marks message boundaries and speaker roles. If you use a different template at inference than during training, the model receives token sequences it was never trained on in those structural positions, causing it to generate incoherent, off-format, or hallucinated output.

3. Compare Alpaca and ShareGPT formats. When would you choose one over the other?

▶ Show Answer
Alpaca uses three fields (instruction, input, output) and is ideal for single-turn instruction tasks where each example is a standalone question-answer pair. ShareGPT uses a list of conversation turns and is better for multi-turn dialogues, where the model needs to maintain context across several exchanges. If you're building a simple Q&A system, use Alpaca. If you're building a conversational assistant, use ShareGPT.

4. Why is dataset quality more important than dataset size in SFT?

▶ Show Answer
In SFT, the model directly imitates the responses in the training data. If responses are inconsistent, poorly written, contain errors, or don't match a coherent style, the model learns those flaws. Because each example is a direct behavioural template, 1,000 high-quality expert-written responses produce a better model than 100,000 mediocre ones scraped without curation. This contrasts with CPT, where sheer volume of domain text helps even with some noise.

5. Why should packing=False be used for SFT but packing=True can be used for CPT?

▶ Show Answer
Packing concatenates multiple training samples into a single long sequence to fill the context window. For CPT (raw text), this is safe because there are no loss masks to preserve. For SFT, each sample has a precise loss mask (prompt tokens masked, response tokens unmasked). Packing can cause samples from different examples to run together in a single context window, making it difficult to correctly apply the loss mask — the boundary between where one example's response ends and another's prompt begins may not be cleanly identifiable, corrupting what the model learns.

6. What does add_generation_prompt=True do in apply_chat_template(), and why is it important?

▶ Show Answer
It appends the assistant's opening header token (e.g. <|start_header_id|>assistant<|end_header_id|> for Llama-3) after the user's message, but before any content. This signals to the model that it should now generate the assistant's response. Without it, the model doesn't know it's expected to generate — it might try to predict more user text, generate nothing, or produce garbled output.

7. In what order do CPT (Class 21) and SFT (Class 22) typically appear in a production LLM pipeline, and why?

▶ Show Answer
CPT comes first, SFT comes second. CPT fills the model with domain knowledge — vocabulary, facts, and writing patterns — by training on raw text. SFT then teaches the model to express that knowledge helpfully in response to instructions. If you did SFT first, the model could follow instructions but might lack the domain expertise to answer correctly. Doing CPT first ensures the knowledge is there; SFT then makes it accessible through conversation.
